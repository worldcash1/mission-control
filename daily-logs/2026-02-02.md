## Browser Port Conflict Fix (AAR)

**Issue:** Browser snapshot/screenshot failed with `EADDRINUSE 127.0.0.1:18792`

**Root cause:** Gateway bound to 18792 for Chrome relay by default. Snapshot tried to reuse same port.

**Fix:** Added explicit browser profile config:
```json
"browser": {
  "profiles": {
    "clawd": { "cdpPort": 18800, "color": "#FF4500" },
    "chrome": { "cdpPort": 18803, "driver": "extension", "color": "#00AA00" }
  }
}
```

**Lesson:** When browser tools fail with port conflicts, check `browser.profiles` in config. Explicit ports prevent gateway conflicts.
